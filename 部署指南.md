# 部署指南 · 合肥拆除 FAQ 卡片

## 方案一：Render（推荐 · 免费 · 最简单）

Render 是一个免费的云托管平台，支持 Node.js，国内可以访问。

### 特点
- ✅ 完全免费
- ✅ 不需要改任何代码
- ✅ 自动部署，更新代码后自动重新发布
- ⚠️ 15分钟没人访问会休眠，首次打开需要等 30 秒左右唤醒

### 部署步骤

#### 第一步：注册 GitHub 账号
1. 打开 https://github.com/signup
2. 用邮箱注册一个账号（国内邮箱可用）
3. 验证邮箱

#### 第二步：把代码上传到 GitHub
1. 登录 GitHub 后，点击右上角 `+` → `New repository`
2. 仓库名称填 `hefei-chaichu-faq`
3. 点击 `Create repository`
4. 在电脑上打开 `chaichu-faq-card` 文件夹
5. 把文件夹里的所有文件（除了 node_modules）打包成 zip
6. 在 GitHub 仓库页面点击 `uploading an existing file`
7. 把 zip 里的文件全部拖进去上传
8. 点击 `Commit changes`

#### 第三步：注册 Render 并部署
1. 打开 https://dashboard.render.com/
2. 点击 `Get Started for Free`，用 GitHub 账号登录
3. 登录后点击 `New +` → `Web Service`
4. 选择你刚创建的 `hefei-chaichu-faq` 仓库
5. 配置如下：
   - Name: `hefei-chaichu-faq`（随便填）
   - Region: `Singapore`（新加坡，离国内最近）
   - Branch: `main`
   - Runtime: `Node`
   - Build Command: 留空
   - Start Command: `node server.js`
   - Plan: `Free`
6. 点击 `Create Web Service`
7. 等待 2-3 分钟，部署完成后会显示一个链接：
   `https://hefei-chaichu-faq.onrender.com`

#### 完成
把这个链接分享到朋友圈即可！

### 更新内容
以后修改了 `faq.json` 或代码，只需要在 GitHub 上传更新后的文件，Render 会自动重新部署，不需要额外操作。

## 方案二：Railway（备选 · 免费）

Railway 也是免费的 Node.js 托管平台，操作类似 Render。

1. 打开 https://railway.app/
2. 用 GitHub 登录
3. 点击 `New Project` → `Deploy from GitHub repo`
4. 选择仓库，直接部署

## 方案三：内网穿透（临时分享 · 无需注册）

如果你只是临时分享给几个人看看，不想注册任何平台，可以用内网穿透工具。

### 使用 cpolar（推荐）
1. 下载 cpolar：https://www.cpolar.com/download
2. 注册账号，获取 authtoken
3. 在电脑上配置并运行：
   ```
   cpolar authtoken xxxxxx
   cpolar http 3000
   ```
4. 会生成一个公网链接，如 `https://abcd.cpolar.top`
5. 把这个链接分享到朋友圈

⚠️ 注意：内网穿透需要你的电脑一直开着，且 cpolar 免费版链接会变化（每次重启不同）。

## 常见问题

**Q: Render 访问慢怎么办？**
A: 免费版首次访问需要唤醒，大概等 30 秒。之后正常速度。如果介意，可以考虑付费版（$7/月）或 Railway。

**Q: 可以绑定自己的域名吗？**
A: Render 支持自定义域名。在 Render 后台 → Settings → Custom Domains 中添加你的域名，然后按提示配置 DNS 即可。

**Q: 后台管理在线上能用吗？**
A: 可以。线上链接后面加上 `/admin.html`，输入密码 `admin123` 就能管理。但线上管理修改的数据只在服务器运行期间有效——如果服务器重启，修改会丢失（因为免费版没有持久化存储）。

**建议**：重要内容修改在本地完成，然后通过 GitHub 更新到线上。
